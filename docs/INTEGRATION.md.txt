# OP721 Reloaded — Integration Guide

This document explains how to integrate **OP721 Reloaded** into:

- Wallets  
- Marketplaces  
- Indexers  
- dApps  
- OP‑20 payment flows  
- OP_NET smart contracts  

OP721 Reloaded is designed to be **fully ERC‑721 compatible**, while using OP_NET’s **WASM/TypeScript‑style contract language**.

---

# 1. Contract Interface Overview

```ts
balanceOf(owner: Address): u64
ownerOf(tokenId: u64): Address

transferFrom(from: Address, to: Address, tokenId: u64): void
safeTransferFrom(from: Address, to: Address, tokenId: u64): void
safeTransferFromWithData(from: Address, to: Address, tokenId: u64, data: Uint8Array): void

approve(to: Address, tokenId: u64): void
setApprovalForAll(operator: Address, approved: bool): void

getApproved(tokenId: u64): Address | null
isApprovedForAll(owner: Address, operator: Address): bool
```

Metadata:

```ts
name(): string
symbol(): string
tokenURI(tokenId: u64): string
```

Receiver interface:

```ts
onOP721Received(operator, from, tokenId, data): string
```

---

# 2. Integrating with Wallets

Wallets must support:

### ✔ Ownership  
```ts
ownerOf(tokenId)
```

### ✔ Balances  
```ts
balanceOf(address)
```

### ✔ Approvals  
- Approve single token  
- Approve operator (setApprovalForAll)

### ✔ Safe transfers  
Default to:

```ts
safeTransferFrom(from, to, tokenId)
```

### ✔ Metadata  
Call:

```ts
tokenURI(tokenId)
```

---

# 3. Integrating with Marketplaces

Marketplaces must support:

### ✔ Ownership verification  
```ts
ownerOf(tokenId)
```

### ✔ Approval checks  
```ts
isApprovedForAll(owner, marketplaceAddress)
getApproved(tokenId)
```

### ✔ Transfer flow  
```ts
transferFrom(seller, buyer, tokenId)
```

Or safe version:

```ts
safeTransferFrom(seller, buyer, tokenId)
```

### ✔ Escrow compatibility  
- OP‑20 tokens for payment  
- NFT stays with seller or moves to escrow contract  
- Escrow releases funds after transfer  

### ✔ Event indexing  
Listen for:

- `Transfer`
- `Approval`
- `ApprovalForAll`

---

# 4. Integrating with Indexers

Indexers should track:

### ✔ Ownership  
`Transfer(from, to, tokenId)`

### ✔ Approvals  
`Approval(owner, approved, tokenId)`  
`ApprovalForAll(owner, operator, approved)`

### ✔ Metadata  
Call:

```ts
tokenURI(tokenId)
```

### ✔ Total supply  
Track mints and burns.

---

# 5. Integrating with OP‑20 Tokens

OP721 Reloaded works seamlessly with OP‑20 tokens for:

- Payments  
- Auctions  
- Escrow  
- Lending  
- Collateralization  

### Example purchase flow

1. Buyer approves marketplace:

```ts
op20.approve(marketplace, price)
```

2. Marketplace moves OP‑20 into escrow:

```ts
op20.transferFrom(buyer, escrow, price)
```

3. Marketplace transfers NFT:

```ts
op721.transferFrom(seller, buyer, tokenId)
```

4. Escrow releases OP‑20 to seller.

---

# 6. Integrating with Smart Contracts

Other OP_NET contracts can:

### ✔ Verify ownership  
```ts
op721.ownerOf(tokenId)
```

### ✔ Transfer NFTs  
```ts
op721.transferFrom(msg.sender, contractAddress, tokenId)
```

### ✔ Receive NFTs  
Implement:

```ts
onOP721Received(operator, from, tokenId, data): string {
  return "OP721_RECEIVED";
}
```

### ✔ Use NFTs as collateral  
Store:

- owner  
- tokenId  
- contractAddress  

Lock transfers until loan is repaid.

---

# 7. Safe Transfer Rules

When using `safeTransferFrom`, the receiver must:

- Be an EOA **or**
- Implement `IOP721Receiver`

If not, the transfer **must revert**.

# 9. Testing Your Integration

Recommended tests:

### ✔ Ownership  
- Mint → ownerOf  
- Transfer → ownerOf changes  
- Burn → ownerOf reverts  

### ✔ Approvals  
- Approve single token  
- Approve operator  
- Revoke operator  

### ✔ Transfers  
- transferFrom  
- safeTransferFrom  
- safeTransferFromWithData  

### ✔ Receiver contract  
- Must return `"OP721_RECEIVED"`

---

# 10. Best Practices

- Always use **safeTransferFrom** unless receiver is known EOA.  
- Always check **isApprovedForAll** before listing NFTs.  
- Always index **Transfer** events.  
- Always validate **tokenURI**.  
- Always verify **ownerOf** before executing marketplace actions.

---

# 11. Compatibility Notes

OP721 Reloaded is:

- Fully compatible with OP_NET  
- Fully compatible with OP‑20  
- Fully compatible with OP721 Marketplace  
- Fully compatible with indexers  
- Fully compatible with wallets  
- Fully compatible with DeFi protocols  

Legacy OP_721 contracts **must use an adapter**.
