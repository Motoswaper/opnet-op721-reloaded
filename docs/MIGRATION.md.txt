# Migration from legacy OP_721 to OP721 Reloaded

The original `OP_721` implementation had two critical issues:

1. `ownerOf` returned a **hash of the owner address**, not the address itself.
2. Some methods required the caller to **pass the owner address** as a parameter.

OP721 Reloaded fixes both:

- `ownerOf` now returns the **real owner address**.
- All methods derive ownership from storage, not from caller input.

## Migration strategies

### New collections

For any new NFT collection on OP_NET:

- Use OP721 Reloaded as the base implementation.
- Do **not** use the legacy OP_721 contract.

### Existing collections

For existing OP_721 collections:

- Option A: **Wrap** them with an adapter contract that:
  - Tracks real owners in a new mapping
  - Exposes the OP721 Reloaded interface
- Option B: **Re‑mint** into a new OP721 Reloaded collection and deprecate the old one.
